1Arquitectura del sistema

MagicMirror² es una aplicación Node.js + Electron que corre un servidor web Express en el puerto 8080. La interfaz se renderiza en Chromium (Electron) a pantalla completa. Todos los módulos son archivos JavaScript que se cargan dinámicamente.

ComponenteTecnologíaFunción
MotorNode.js 22 + ElectronProceso principal, ventana fullscreen
ServidorExpress + Socket.ioHTTP en :8080, comunicación en tiempo real
FrontendHTML/CSS/JS vanillaRenderizado de módulos en pantalla
MódulosJS + node_helper.jsLógica de cada widget
PáginasMMM-pagesSistema de navegación entre páginas
Control remotoMMM-Remote-ControlAPI REST + interfaz web

Accesos de red — URLs completas

Este proyecto tiene dos servidores activos a la vez en el mismo ordenador:

PuertoQuién lo sirvePara qué
:8080MagicMirror² (Express + Node.js)La app del espejo, su API y todos sus archivos de módulos
:5500VS Code Live Server (extensión)Sirve los archivos del proyecto directamente desde el disco para previsualización en el navegador durante el desarrollo
ℹ ¿Por qué PAGINAS.html funciona en ambos puertos? PAGINAS.html es un archivo HTML estático con JavaScript. Cuando lo abre VS Code Live Server en :5500, el HTML se carga desde tu disco — pero los botones del panel llaman a fetch('http://192.168.18.112:8080/api/...'), que sí va al servidor de MagicMirror. Así el panel de control funciona desde cualquier servidor que sirva el HTML, siempre que MagicMirror esté corriendo en :8080.
⚠ Cómo verificar la conexión PAGINAS.html incluye un botón "🔍 Probar conexión". Al pulsarlo, hace una llamada GET a http://192.168.18.112:8080/api/. Si MagicMirror responde → muestra "Conexión OK". Si no → muestra el error exacto (Failed to fetch, HTTP 401, etc.). Úsalo siempre que tengas dudas sobre si el espejo está activo.

3Módulos configurados

A continuación se describe cada módulo activo: qué hace, cómo está configurado, cómo verificar que funciona y qué hacer si falla.

🔒 Módulos fijos (siempre visibles)

🔔alertFijoNativo

Qué hace

Sistema de notificaciones emergentes de MagicMirror². Muestra mensajes de alerta en pantalla completa cuando otros módulos lo solicitan. No tiene interfaz visual permanente.

Configuración

Sin parámetros. Se activa simplemente incluyéndolo en modules[].

Verificar

Funciona si MagicMirror arranca sin errores. No produce ninguna salida visual a menos que otro módulo lo use.

🎛MMM-Remote-ControlFijoCustom

Qué hace

Expone un panel de control web y una API REST para gestionar MagicMirror desde cualquier dispositivo de la red: encender/apagar módulos, cambiar config, reiniciar, y enviar notificaciones a los módulos.

Accesos

Panelhttp://192.168.18.112:8080/remote.html
Páginashttp://192.168.18.112:8080/modules/MMM-Remote-Control/PAGINAS.html
APIhttp://192.168.18.112:8080/api/

Configuración

ParámetroValorDescripción
showModuleApiMenutrueMenú de control por módulo en la interfaz web
secureEndpointsfalseAPI sin autenticación (recomendado en red local)

Verificar

Abre http://192.168.18.112:8080/remote.html en el navegador. Debe cargar el panel de control. Usa el botón "🔍 Probar conexión" de PAGINAS.html para verificar que la API responde.

Problemas

⚠ PAGINAS.html muestra error de conexión Usa el botón "🔍 Probar conexión" para ver el error exacto. Si muestra Error de red: Failed to fetch → la API no responde o hay un problema de red. Si muestra HTTP 401 → activa secureEndpoints: false en config.js. Comprueba también que remote.html carga correctamente.
🎂MMM-CardDavBirthdaysProviderFijoCustom

Qué hace

Se conecta a un servidor CardDAV (Nextcloud) y extrae los cumpleaños de los contactos. Genera un archivo de calendario iCal accesible localmente, que el módulo calendar consume.

Configuración

ParámetroValor configuradoDescripción
hosthttps://gio.it.tab.digital/remote.php/davURL base del servidor CardDAV
usernameespejomagic12@gmail.comUsuario del servidor
password(configurada)Contraseña del servidor
calendarNameCumpleNombre exacto de la agenda en el servidor
ℹ Endpoint generadoCuando el módulo conecta correctamente, crea: http://192.168.18.112:8080/MMM-CardDavBirthdaysProvider.ics

Verificar

curl http://192.168.18.112:8080/MMM-CardDavBirthdaysProvider.ics

Debe devolver texto con BEGIN:VCALENDAR. Si devuelve 404 o no responde → el módulo no pudo conectar al servidor CardDAV.

Problemas

⚠ Error en el módulo calendar / "consulta los registros" Este error significa que calendar no puede obtener el archivo .ics, normalmente porque MMM-CardDavBirthdaysProvider no pudo conectar al servidor CardDAV. Pasos:
1. Verifica que el servidor gio.it.tab.digital es accesible: curl -I https://gio.it.tab.digital
2. Verifica que el nombre de la agenda (calendarName: "Cumple") coincide exactamente con el nombre en Nextcloud.
3. Prueba las credenciales en https://gio.it.tab.digital desde el navegador.
4. Consulta los logs de MagicMirror: npm start 2>&1 | grep -i carddav

💬complimentsPágina 1Nativo

Qué hace

Muestra frases motivacionales en el tercio inferior de la pantalla. Las frases cambian según la franja horaria: mañana (5–12h), tarde (12–20h), noche (resto). Cada frase es visible 30 segundos.

Verificar

Qué hace

Muestra la previsión meteorológica de Valencia: temperatura actual, máx/mín, humedad, viento, amanecer/atardecer, previsión por horas y por días. Usa la One Call API 3.0 de OpenWeather.

Configuración

ParámetroValorDescripción

Verificar

# Si responde {"cod":401} → key inactiva o sin suscripción

2. Ve a openweathermap.org/subscriptions → activa One Call API 3.0 (gratuita, puede requerir tarjeta).
3. Las nuevas activaciones pueden tardar hasta 2 horas. Reinicia MagicMirror tras activar.

⚙ Controlador del sistema

📄MMM-pagesControladorCustom

Qué hace

Gestiona el sistema de páginas: controla qué módulos son visibles en cada página y cuáles son siempre visibles. No tiene interfaz visual propia. Responde a las notificaciones PAGE_CHANGED, PAGE_INCREMENT y PAGE_DECREMENT.

Configuración clave

ParámetroValorDescripción
modules[[p1...],[p2...],[p3...]]Array de páginas; cada página es un array de nombres de módulos

homePage0Índice de la página de arranque (0 = primera)

Añadir una página nueva

En modules[] añade un nuevo array: ["NombreModulo"]. Actualiza pages: 3 en MMM-Page-Indicator al nuevo número.

Problemas

Un módulo aparece en la página incorrecta → El nombre en modules[] no coincide exactamente con el campo module: en config.js (sensible a mayúsculas).

4Control de páginas desde el navegador

Método A — Panel visual (recomendado)

Abre esta URL en cualquier navegador o tablet de tu red:

Panelhttp://192.168.18.112:8080/modules/MMM-Remote-Control/PAGINAS.html

Verás tres botones grandes (Inicio, Finanzas & Música, Contenido Web) y dos flechas para avanzar/retroceder. Al pulsar un botón, el espejo cambia de página inmediatamente y el botón se ilumina.

Método B — API REST directa

Útil para automatizaciones, scripts o botones en otras páginas web.

# Ir a la Página 1 (Inicio)
curl -X POST http://192.168.18.112:8080/api/notification/PAGE_CHANGED \
     -H "Content-Type: application/json" \
     -d '{"payload": 0}'

# Ir a la Página 2 (Finanzas)
curl -X POST http://192.168.18.112:8080/api/notification/PAGE_CHANGED \
     -H "Content-Type: application/json" \
     -d '{"payload": 1}'

# Ir a la Página 3 (Web)
curl -X POST http://192.168.18.112:8080/api/notification/PAGE_CHANGED \
     -H "Content-Type: application/json" \
     -d '{"payload": 2}'

# Avanzar una página
curl -X POST http://192.168.18.112:8080/api/notification/PAGE_INCREMENT

# Retroceder una página
curl -X POST http://192.168.18.112:8080/api/notification/PAGE_DECREMENT

Método C — Botón HTML en otra página web

<!-- Pega esto en cualquier HTML de tu red -->
<button onclick="cambiarPagina(1)">🎵 Ir a Spotify</button>

<script>
function cambiarPagina(indice) {
  fetch('http://192.168.18.112:8080/api/notification/PAGE_CHANGED', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ payload: indice })
  });
}
</script>

Referencia de notificaciones de página

NotificaciónPayloadEfecto
PAGE_CHANGED0Ir a Página 1 (Inicio)
PAGE_CHANGED1Ir a Página 2 (Finanzas)
PAGE_CHANGED2Ir a Página 3 (Web)
PAGE_INCREMENT—Siguiente página (cíclico)
PAGE_DECREMENT—Página anterior (cíclico)

Activar rotación automática de páginas

Localiza el bloque de MMM-pages en config/config.js y cambia el valor default: 0 dentro de timings:

// Sin rotación (manual):
timings: { default: 0 }

// Rotar cada 30 segundos:
timings: { default: 30000 }

// Rotación distinta por página (Página 3 dura más):
timings: {
  default: 20000,  // 20 segundos para páginas 1 y 2
  2: 60000        // 60 segundos para la página 3 (índice 2)
}

5Autenticación de Spotify

🔑 Obligatorio antes del primer uso Sin este paso el módulo Spotify aparece en blanco. Solo se hace una vez. El token se renueva automáticamente a partir de entonces.
  1. Verificar el archivo de credenciales

    Comprueba que existe y tiene contenido:

    cat ~/MagicMirror/modules/MMM-Spotify/spotify.config.json

    Debe mostrar tu CLIENT_ID, CLIENT_SECRET y REDIRECT_URI: http://192.168.18.112:8888/callback.

  2. Registrar la Redirect URI en Spotify Developer Dashboard
    URLhttps://developer.spotify.com/dashboard

    Inicia sesión → selecciona tu aplicación → Edit Settings → Redirect URIs → añade:

    http://192.168.18.112:8888/callback

    Guarda los cambios.

  3. Ejecutar el script de autenticación 
    cd ~/MagicMirror/modules/MMM-Spotify
node first_auth.js

    El script mostrará una URL. Ábrela en el navegador o la abrirá automáticamente.

  4. Autorizar en el navegador

    Spotify pedirá permiso. Haz clic en Acepto. Serás redirigido a 192.168.18.112:8888/callback. La página puede aparecer vacía — es normal.

  5. Verificar que se creó el token 
    ls ~/MagicMirror/modules/MMM-Spotify/*token*.json

    Debe aparecer un archivo MagicMirror_token.json. Si existe, la autenticación fue correcta.

  6. Arrancar MagicMirror 
    cd ~/MagicMirror && npm start

    Navega a la Página 2. Spotify mostrará la reproducción en curso (si hay algo sonando en tu cuenta).

💡 Spotify en modo idle Si no hay música reproduciéndose, el módulo muestra el logo de Spotify o aparece en blanco. Es normal. En cuanto empiece a sonar algo en cualquier dispositivo de tu cuenta Spotify, aparecerá automáticamente.

6API Key del tiempo (OpenWeather)

⚠ La One Call API 3.0 NO es automáticamente gratuita Desde 2022, OpenWeather requiere registrar una tarjeta de crédito para activar la One Call API 3.0, incluso con el plan gratuito (1.000 llamadas/día son suficientes y no se cobra si no se supera el límite).
  1. Comprobar si la key actual está activa 
    curl "https://api.openweathermap.org/data/3.0/onecall?lat=39.4699&lon=-0.3763&appid=ccb17f609538b65003669414418539a0"

    Si responde con JSON meteorológico → la key funciona. Si responde "cod": 401 → la key está inactiva.

  2. Si la key no funciona: activarla o crear una nueva
    URLhttps://home.openweathermap.org/subscriptions

    Inicia sesión con tu cuenta de OpenWeather → Subscriptions → activa One Call API 3.0 (plan gratuito). Puede requerir añadir una tarjeta de crédito.

  3. Obtener una nueva API key (si no tienes cuenta)
    URLhttps://home.openweathermap.org/users/sign_up

    Tras registrarte, ve a API keys y copia la key generada. Las nuevas keys pueden tardar hasta 2 horas en activarse.

  4. Actualizar la key en config.js

    Abre config/config.js y localiza la línea:

    apikey: "ccb17f609538b65003669414418539a0",

    Sustitúyela por tu nueva key.

           display: flex; align-items: center; justify-content: center; overflow: hidden; }

<body>

  <script>
    const src = new URLSearchParams(location.search).get('src')

EOF

6 · Configurar credenciales de Spotify

Primero necesitas crear una aplicación en el Spotify Developer Dashboard:

  1. Crear app en Spotify Developer

    Ve a developer.spotify.com/dashboard → Create App.
    Rellena: Name, Description, Redirect URI: http://127.0.0.1:8888/callback → Guarda.

  2. Obtener CLIENT_ID y CLIENT_SECRET

    En la app creada → Settings. Copia el Client ID y el Client Secret.

  3. Crear spotify.config.json 
    cat > ~/MagicMirror/modules/MMM-Spotify/spotify.config.json << 'EOF'
[
  {
    "USERNAME":      "MagicMirror",
    "CLIENT_ID":     "TU_CLIENT_ID_AQUI",
    "CLIENT_SECRET": "TU_CLIENT_SECRET_AQUI",
    "TOKEN":         "./MagicMirror_token.json",
    "AUTH_DOMAIN":   "http://localhost",
    "AUTH_PORT":     "8888"
  }
]
EOF
  4. Autenticar (solo primera vez) — ejecutar en Windows

    El script auth_windows.js abre un servidor local en el puerto 8888 y espera el callback de Spotify.

    # En Windows PowerShell o CMD, dentro de la carpeta del módulo:
cd MagicMirror\modules\MMM-Spotify
node auth_windows.js

    El script mostrará una URL larga. Ábrela en el navegador, acepta los permisos de Spotify y espera el mensaje "✓ Token guardado".

  5. Verificar que se creó el token 
    ls ~/MagicMirror/modules/MMM-Spotify/*token*.json
# Debe aparecer: MagicMirror_token.json

7 · Obtener API Key de OpenWeather (gratuita, sin tarjeta)

  1. Crear cuenta gratuita

    Regístrate en home.openweathermap.org/users/sign_up.
    Ve a My API Keys y copia la key generada automáticamente.

  2. Esperar activación

    Las nuevas keys pueden tardar hasta 2 horas en activarse. Verifica así:

    curl "https://api.openweathermap.org/data/2.5/weather?q=Valencia,ES&appid=TU_API_KEY&units=metric"
# Si responde JSON con temperatura → key activa y lista
# Si responde {"cod":401} → espera unos minutos más
  3. Nota importante: usar data/2.5, NO data/3.0

    Los endpoints data/2.5/weather y data/2.5/forecast son 100% gratuitos y no requieren tarjeta. El módulo nativo weather de MagicMirror los soporta directamente con apiVersion: "2.5".

    ⚠ No uses data/3.0/onecall — Requiere suscripción con tarjeta aunque sea gratuita. Quédate en apiVersion: "2.5".
